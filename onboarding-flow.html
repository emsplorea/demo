03
Send Menu Master Data
Batch · Initial Sync + Delta Updates
› details
Batch · Webhook ⚿ Idempotency: externalItemId per entity
Purpose
Push full menu catalog: groups → categories → items → attributes
Initial Sync: full catalog push (all entities, idempotent upsert)
Delta Updates: Upsert · Deactivate events per entity (no hard delete)
Endpoint + Payload
POST /v1/integrations/masterdata/menu { "locationId": "loc_47", "syncType": "initial", // initial | delta "groups": [ { "externalGroupId": "g_10", "name": "Burgers" } ], "categories": [ { "externalCategoryId": "cat_20", "name": "Classic", "groupIds": ["g_10"] } ], "items": [ { "externalItemId": "item_301", "name": "Cheeseburger", "categoryIds": ["cat_20"], "price": { "amount": 149, "currency": "NOK" }, "attributes": [ { "type": "addon", "name": "Extra cheese", "price": 29 }, { "type": "variant", "name": "Large", "price": 179 } ], "active": true } ] } Natural keys: externalGroupId, externalCategoryId, externalItemId
DB Canonical Mapping
Group → menus_groups.id Group↔Menu → menus_groups_menus (join table) Category → menus_categories.id Item → menus_items.id Item↔Category → menus_items_categories (join table, multi-cat) Attribute → menus_items_attributes.id type: addon | variant | modifier
Delta Event Types
Upsert → create or update (matched on external ID) Deactivate → set active=false (soft delete, never hard delete) ⚠ Delete → not supported — use Deactivate instead

Governance Rules
Onboarding Governance
Idempotent upsert on all endpoints — re-push is always safe
External IDs are natural keys — never rely on Plorea internal IDs externally
No hard deletes — only soft deactivation via active=false
Role matrix required — external roles never trusted directly
Sync status gates ordering — PARTIAL/FAILED blocks ordering surfaces
Catalog versioning — catalogHash + version tracked per sync
Integration mode enforced — financial surfaces disabled on activation
🚫 What We Refuse To Do
✕ No financial reporting in Plorea when integration active
✕ No hard deletes — deactivation only, always recoverable
✕ No direct role passthrough — always mapped via matrix
✕ No ordering before syncStatus = OK
✕ No Plorea-internal IDs exposed to 3rd party
